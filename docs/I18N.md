# Internationalization (frontend)

Supported languages:

| Language        | Code    | Notes                          |
|----------------|---------|--------------------------------|
| Thai           | `th`    | Default + fallback             |
| English        | `en`    |                                |
| Chinese (Simplified) | `zh-CN` | Noto Sans SC loaded in `index.html` |
| Malay (Bahasa Malaysia) | `ms` |                                |

## Implementation

- **Libraries:** `i18next`, `react-i18next`, `i18next-browser-languagedetector`
- **Bootstrap:** `src/i18n/index.ts` (imported from `src/main.tsx`)
- **Translation files:** `src/i18n/locales/{th,en,zh-CN,ms}.json`
- **Persistence:** `localStorage` key `gamewallet_lang` (see `i18n` `detection.lookupLocalStorage`)
- **HTML lang / document title:** `src/components/I18nSync.tsx` (mounted in `App.tsx`)
- **Day.js:** locale follows app language in `I18nSync`
- **Number formatting:** `src/i18n/useAppLocale.ts` (`Intl` / `toLocaleString` locale)
- **Language switcher:** `src/components/LanguageSwitcher.tsx` (in customer header, admin bar, agent bar, login/register)

## Adding strings

1. Add the key under the same path in all four JSON files.
2. Use `useTranslation()` and `t('key.subkey')` in components.
3. Admin sidebar labels use `labelKey` in `src/config/adminMenuKeys.ts` (e.g. `admin.menu.orders`).

## Not yet fully translated

Some secondary pages (e.g. certain customer product/order detail screens, analytics copy) may still mix English; extend the JSON files and replace hard-coded strings as needed.
